# MATH-001 数学证明文档

## 问题定义

**RISK-H-003**: 候选检索爆炸问题
- 高维向量（>768维）场景下，候选集可能无限增长
- TB级数据（1e8 chunks）时，候选检索次数可达 2.081e11次

## 数学原理

### 1. 鸽巢原理应用

**定理**: 对于汉明距离 d < threshold 的两个SimHash，当分段数为 s 时，至少存在 (s - d) 个相同段。

**证明**:
- 汉明距离 d 表示有 d 个位不同
- 每个不同的位最多影响 1 个段
- 因此最多影响 d 个段
- 至少存在 s - d 个段完全相同 ∎

**推论** (假阴性率 = 0%):
- 当 s = 4, d < 3 时，至少 2 个段相同
- 查询时的并集操作一定能捕获相似项

### 2. 冲突概率计算

对于64位SimHash，均匀分布假设下：

```
P(d < 3) = sum_{k=0}^{2} C(64,k) / 2^64
         = (C(64,0) + C(64,1) + C(64,2)) / 2^64
         = (1 + 64 + 2016) / 1.84e19
         ≈ 2.08e3 / 1.84e19
         ≈ 1.13e-16
```

对于 N 个向量，期望候选数：
```
E[|C|] = N * P(d < 3)
```

| N | E[|C|] |
|---|--------|
| 1e6 | 1.13e-10 |
| 1e8 (TB级) | 1.13e-8 |
| 1e10 | 1.13e-6 |

### 3. 高维场景 (>768维)

当维度 > 768 时：
- 概率计算：P(d < 3) 随维度指数下降
- 实际风险：数据相关性导致实际冲突率远高于理论值
- 解决：硬限制候选集大小为 100

### 4. 复杂度对比

| 方法 | 单次查询复杂度 | 1e8 chunks总查询 |
|------|---------------|------------------|
| 暴力枚举 | O(2081) | 2.081e11 |
| LSH-4段 | O(4) | 4e8 |
| **加速比** | **520x** | **520x** |

## 实现验证

### 候选限制器 (CandidateLimiter)

```typescript
limitCandidates(candidates, limit = 100)
```

- 输入: 候选列表 [(id, distance)]
- 输出: 前 limit 个最近候选
- 时间复杂度: O(N log N)

### 溢出保护 (OverflowProtectedCalculator)

```typescript
safeDistance(a: bigint[], b: bigint[]): number
```

- 检测: BigInt乘法溢出
- 处理: 返回 Number.MAX_SAFE_INTEGER
- 保证: 计算不崩溃

## 结论

MATH-001 通过以下机制解决候选检索爆炸：

1. **鸽巢原理保证**: LSH索引无假阴性
2. **概率上界**: 理论冲突率 < 1e-16
3. **硬限制**: 候选集上限 100
4. **溢出保护**: 高维向量安全计算
